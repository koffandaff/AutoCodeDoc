Statistics (mean and standard deviation) are computed over the training set only and stored as attributes in the .zarr store. The same statistics are applied during validation and testing to prevent data leakage.

Data Augmentation

Currently, the neural-lam pipeline uses no spatial augmentation (weather data has fixed coordinate grids). Temporal augmentation is achieved through:

  • Random windowing: Different start times within training period

  • Multi-step rollout: Variable forecast horizon during training

Configuration Example

# mllam-data-prep config.yaml
dataset:
  name: danra_example
  source: /data/danra/
  variables:
    surface:
      - 2m_temperature
      - 10m_u_component_of_wind
      - 10m_v_component_of_wind
      - mean_sea_level_pressure
    atmosphere:
      levels: [500, 700, 850, 1000]
      vars:
        - temperature
        - geopotential
  grid:
    type: regular_ll
    resolution: 0.25
  splits:
    train: ["1990-01-01", "2015-12-31"]
    val: ["2016-01-01", "2017-12-31"]
    test: ["2018-01-01", "2020-12-31"]
